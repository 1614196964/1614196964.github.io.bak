软件工程复习指南

软件工程
发布日期:   2021-12-02

软件工程七大本质特征

软件工程是指导计算机软件开发和维护的一门工程学科.

采用工程的概念、原理、技术和方法来开发与维护软件

把经过时间考验证明正确的管理技术和当前最好的技术方法结合起来

经济地开发出高质量的软件并有效地维护它

  1. 软件工程关注于大型项目的构造
  2. 软件工程的中心课题是控制复杂性
  3. 软件经常变化
  4. 软件的开发效率非常重要
  5. 和谐的合作是软件开发的关键
  6. 软件必须有效地支持它的用户
  7. 在软件工程领域中通常由具有一种文化背景的人替具有另一种文化背景的人创造产品

忽视会导致软件开发的失败

软件生命周期

  1. 问题定义
  2. 可行性分析
  3. 需求分析
  4. 总体设计
  5. 详细设计
  6. 编码和单元测试
  7. 综合测试
  8. 软件维护

瀑布模型

瀑布模型是基于文档驱动的模型

各个阶段产生的文档是维护软件必不可少的, 没有文档的软件几乎是不可维护的

特点:

  1. 阶段间具有顺序性和依赖性
  2. 推迟实现的观点
  3. 质量保证的观点

可行性研究

  1. 目的: 用最小的代价在尽可能短的时间内确定问题能否解决
  2. 任务: 不是解决问题, 是确定问题能否被解决
  3. 可行性
    1. 技术可行性
    2. 经济可行性
    3. 操作可行性
    4. 法律可行性
    5. 社会效益可行性

数据流图

    1. 信息交流的工具
    2. 分析和设计的工具

数据字典

  1. 内容

    1. 数据流
    2. 数据流分量(数据元素)
    3. 数据存储
    4. 处理

  2. 用途

    2. 控制信息, 便于集成测试
    3. 数据库开发的第一步

与用户沟通获取需求的方法

  1. 访谈
  2. 面向数据流图的自顶向下求精
  3. 简易的应用规格说明技术
  4. 快速建立软件原型
    • 最准确, 最有效, 最强大的需求分析技术

规格说明

通过需求分析处理创建分析模型之外, 还应该写出软件需求规格说明书

软件需求规格说明书是需求分析阶段的出来的最主要的文档

总体设计原理

  1. 模块化
    1. 把程序划分成独立命名且可独立访问的模块
    2. 每个模块完成一个子功能
    3. 把这些模块集合起来构成整体可以满足用户的需求
    4. 模块不是越细越好, 因为还有接口成本呢, 模块多了, 接口也变多了
  2. 抽象
    1. 最高层次: 用问题环境的语言, 以概括的方式叙述问题的解法(软件定义)
    2. 较低层次: 软件设计
    3. 最低层次: 软件实现
  3. 逐步求精
  4. 信息隐藏和局部化
    1. 信息隐藏: 模块内包含的信息, 对不需要这些信息的模块来说是不能访问的
    2. 局部化: 把一些关系密切的软件元素物理的放得彼此靠近
    3. 局部化有助于信息隐藏
  5. 模块独立
    1. 容易开发
    2. 容易测试和维护

衡量模块独立的标准

耦合

耦合是一个软件结构内不同模块之间互连程度的度量

一个结构, 不同模块的互连程序

耦合的强弱取决于:

  1. 模块间接口的复杂程度
  2. 通过接口的数据
  3. 进入或访问一个模块的点
  • 尽量使用数据耦合
  • 少用控制耦合和特征耦合
  • 限制公共环境耦合的范围
  • 完全不用内容耦合

内聚(高内聚往往意味着低耦合)

内聚是一个模块内各个元素彼此结合的紧密程度

一个模块, 不同元素的紧密程度

  • 低内聚
    • 偶然内聚
    • 逻辑内聚
    • 时间内聚
  • 中内聚
    • 过程内聚
    • 通信内聚
  • 高内聚
    • 顺序内聚
    • 功能内聚

面向数据流的设计方法

  • 变换流
  • 事务流

PAD图

  1. 优点:
    1. 基于PAD图设计的程序必然是结构化程序
    2. 所描绘的结构十分清晰
    3. 表现的逻辑易读易懂易记
    4. 容易将PAD图转换成高级语言程序
    5. 既可用于表示程序逻辑, 也可用于描绘数据结构
    6. 支持自顶向下, 逐步求精方法的使用

软件测试的目标

尽可能多的发现并排除软件中潜藏的错误最终把一个高质量的软件系统交给用户

  1. 测试是为了发现程序中的错误而执行的过程
  2. 好的测试方案是极可能发现迄今为止尚未发现的错误的测试方案
  3. 成功的测试是发现了至今为止尚未发现的错误的测试

测试不能证明程序中没有错误

穷举测试是不可能的

白盒测试

逻辑覆盖

  1. 语句覆盖
  2. 判定覆盖
    • 每个分支都执行一次
  3. 条件覆盖
    • 每个条件都执行一次
  4. 判定/条件副高
  5. 条件组合覆盖

黑盒测试

  1. 等价划分
  2. 边界值分析
